# Model Card

## Intended Use

GreenAmpt-Nash-Muskingum is a lumped rainfall-runoff and flood-routing research prototype for teaching, exploratory calibration workflows, and secondary development.

It should not be presented as a professionally validated operational flood forecasting system.

## Model Chain

```text
rainfall intensity -> Green-Ampt excess rainfall -> Nash unit hydrograph -> Muskingum routing -> discharge
```

## Parameters

| Submodel | Parameter | Unit | Meaning |
|---|---|---:|---|
| Green-Ampt | `Ks` | m/s | Saturated hydraulic conductivity. |
| Green-Ampt | `suction_head` | m | Wetting-front suction head. |
| Green-Ampt | `delta_theta` | - | Soil water content deficit. |
| Nash | `n` | - | Number/shape parameter for linear reservoirs. |
| Nash | `k` | h | Storage constant. |
| Muskingum | `K_musk` | h | Routing storage/travel-time parameter. |
| Muskingum | `x_musk` | - | Routing weighting factor, normally 0 to 0.5. |

Muskingum routing rejects parameter combinations that do not satisfy `2*K*x <= dt <= 2*K*(1-x)`.

## Validation Boundary

The included example data is synthetic and generated by the model with random noise. It can support smoke tests and demonstrations, but not real watershed claims.

For a real study, require:

- traceable rainfall and discharge observations
- documented catchment area and time step
- physically defensible parameter bounds
- separated calibration and validation periods
- metrics reported for both periods
- explicit statement of limitations

## Public Positioning

Use:

- "Includes WOA calibration, train/test diagnostics, and example data"

- "engineering-certified model"
